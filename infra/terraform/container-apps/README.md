# Terraform: Azure Container Apps

This module provisions a reviewable Container Apps deployment for the platform:

- resource group, Log Analytics workspace, Application Insights resource, and managed Container Apps environment
- shared runtime user-assigned identity for Key Vault secret resolution and optional ACR pulls
- shared private Blob container for model artifacts accessed through managed identity
- API workloads for gateway, application, feature, scoring, decision, assistant, MLOps, and audit
- background worker workloads for broker consumers
- dedicated outbox relay workloads for async publication
- managed OpenTelemetry wiring from Container Apps workloads into Application Insights traces

Production secrets are expected to live in a separately governed Azure Key Vault. This module consumes versionless secret IDs through `key_vault_secret_ids` and grants the shared runtime identity `Key Vault Secrets User` access on `key_vault_id`.

The deployment now treats model-signing material as first-class runtime secrets:

- `mlops` requires `MODEL_SIGNING_PRIVATE_KEY_PEM`
- `scoring` API and worker require `MODEL_SIGNING_PUBLIC_KEY_PEM`
- production posture rejects deployments that omit either signing secret

Scoring and MLOps workloads use `ARTIFACT_STORAGE_BACKEND=azure_blob` with the shared runtime identity granted `Storage Blob Data Contributor` on the artifact storage account. The deployment intentionally avoids Azure Files mounts and storage account access keys in the production artifact path.

Observability posture is also enforced in the module:

- `OTEL_ENABLED=true` is injected for all workloads by default
- the managed Container Apps OpenTelemetry agent exports traces to the workspace-backed Application Insights resource
- applications do not hardcode OTLP endpoint or protocol values in Terraform; the managed environment injects them
- logs continue to land in Log Analytics through the managed environment

Deployment inputs are pinned per service via `service_image_references`, which must contain digest-pinned OCI references generated by the signed image workflow artifact `container-apps.auto.tfvars.json`.

Raw secret variables remain available for non-production bootstrap only. Production guardrails reject direct Terraform-injected DSNs, broker credentials, auth client secrets, model-signing keys, shared auth secrets, or registry passwords.
